# Contributing

Thank you for your interest in contributing to Ada Anvil projects.

## Reporting Issues

This repository is the central helpdesk for all Ada Anvil products. When opening an issue, please include:

- The product or service you are using.
- A clear description of the problem or question.
- Steps to reproduce the issue, if applicable.
- Any relevant logs, screenshots, or error messages.

For open source projects, you can also open issues directly in their respective repositories.

## Contributing to Open Source Projects

If you would like to contribute code to one of our open source projects:

1. Fork the repository.
2. Create a branch for your change.
3. Make your changes, keeping commits focused and well described.
4. Run existing tests and ensure they pass.
5. Open a pull request against the main branch with a clear description of your changes.

## Security

If you discover a security vulnerability, **do not open a public issue**. Please email [security@ada-anvil.io](mailto:security@ada-anvil.io) instead. See [SECURITY.md](SECURITY.md) for details.

## Code of Conduct

All participants are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md).
